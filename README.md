# 国际象棋小课堂 ♟️

面向 4-6 岁儿童的国际象棋启蒙 Web 应用。通过互动课程、人机对弈、趣味小游戏，帮助零基础的孩子认识棋子、学习规则，在激励系统中持续进步。

## 功能

### 📚 互动课程（12课）

循序渐进的课程体系——从认识棋子起步，逐步掌握每种棋子的走法、将军与将杀，最终学会王车易位等特殊技巧。

| 课次 | 标题 | 内容 |
|------|------|------|
| 1 | 认识棋子 | 国王、皇后、车、象、马、兵 |
| 2 | 认识棋盘 | 8×8 格子、行列命名、a1-h8 |
| 3 | 小兵向前走 | 兵的走法、第一步可走两格 |
| 4 | 跳跃的小马 | L 形跳跃，横二竖一 |
| 5 | 斜线主教 | 斜线行走，不限距离 |
| 6 | 直线城堡 | 横竖行走，不限距离 |
| 7 | 强大的皇后 | 皇后 = 车 + 象 |
| 8 | 保护国王 | 国王每次一格，最重要 |
| 9 | 小心——将军！ | 将军的概念和应对 |
| 10 | 你赢了——将杀！ | 将杀判定和胜利条件 |
| 11 | 吃掉棋子 | 攻击、保护和吃子 |
| 12 | 特殊技巧 | 王车易位、吃过路兵、升变 |

每节课包含多步棋盘演示和互动操作，采用"展示→讲解→操作→庆祝"四步教学模式。

### ⚔️ 人机对弈

- **四个难度**：新手 / 简单 / 中等 / 困难
- **可选颜色**：执白（先走）或执黑
- **AI 引擎**：Minimax + Alpha-Beta 剪枝，各难度在搜索深度和评估噪声上有区分，适合儿童逐步提升
- **辅助功能**：走棋提示、悔棋、认输

### 🎮 趣味小游戏（5个）

| 游戏 | 说明 | 计时 |
|------|------|------|
| 🎣 棋子接接乐 | 只点击提示的那种棋子，锻炼棋子识别 | 60秒 |
| 🔨 格子拍一拍 | 打地鼠式快速点击格子 | 45秒 |
| 🐴 迷宫骑士 | 用小马 L 形跳跃穿过棋盘 | 不限时 |
| 🎨 图案复刻 | 照着目标摆棋子，培养棋盘视觉 | 不限时 |
| ⚔️ 吃子任务 | 找到并吃掉最有价值的棋子 | 30秒 |

### 🏆 进度激励系统

- **腰带等级**：白带→黄带→橙带→绿带→蓝带→紫带→红带→棕带→黑带→彩虹带，累计星星自动升级
- **星星评分**：每课/每局游戏 1-3 颗星，基于准确率和提示使用次数
- **徽章收集**：学习、对弈、游戏、连续天数四类共 16 枚徽章
- **连续天数追踪**：每天回来玩自动记录

### 👤 多角色支持

- 可在设置中添加多个角色（适合多孩家庭）
- 每个角色独立进度、星星和徽章
- 表情符号头像选择，无需文字输入

### 🎨 儿童友好设计

- 72px+ 触摸目标，完美适配 iPad 和手机
- 大量表情符号和图标的图形界面，极简文字
- 温柔纠错：答错只有轻柔动画提示，不显示红色叉号
- 动画和音效增加趣味性
- 适配 `prefers-reduced-motion` 无障碍偏好

## 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| 框架 | React 19 + TypeScript | UI 构建 |
| 构建 | Vite | 开发服务器和打包 |
| 路由 | React Router v7 | 页面导航 |
| 状态管理 | Zustand + persist | 全局状态，localStorage 持久化 |
| 国际象棋 | chess.js | 走法生成、规则校验、局面判断 |
| 棋盘渲染 | react-chessboard v5 | 可交互的棋盘组件 |
| 动画 | Motion (Framer Motion) | 页面过渡和交互动画 |
| 音效 | Howler.js | 走子、吃子、正确、错误等音效 |
| 样式 | Tailwind CSS 4 | 响应式布局和主题 |
| 图标 | Lucide React | 导航和按钮图标 |

纯客户端应用，无需后端服务，所有数据通过 localStorage 持久化存储。

## 使用方法

### 安装和运行

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 浏览器访问 http://localhost:5173
```

### 构建生产版本

```bash
npm run build     # TypeScript 检查 + Vite 构建，输出到 dist/
npm run preview   # 本地预览构建结果
```

### 部署到 GitHub 云端

项目已配置为通过 GitHub Pages 自动部署：

1. 在仓库 **Settings → Pages** 中将 **Source** 设置为 **GitHub Actions**。
2. 将代码推送到 `main` 分支后，GitHub 会自动执行 `.github/workflows/deploy-pages.yml`。
3. 部署完成后，可通过仓库的 GitHub Pages 地址访问应用。

由于站点使用 Hash 路由和相对静态资源路径，部署到仓库子路径后也能正常刷新和访问页面。

### 典型使用流程

1. **首次访问**：应用自动创建默认角色「小棋手」。可在设置中修改名字和头像。
2. **学习**：进入「学习」页，按顺序完成 12 节课程（每完成一课解锁下一课）。
3. **对弈**：进入「对弈」页，选择难度和颜色，与 AI 下一盘完整的棋。
4. **游戏**：进入「游戏」页，选择小游戏在玩乐中巩固棋力。
5. **进度**：在「进度」页查看腰带等级、星星总数、徽章收集和连续天数。
6. **多角色**：在「设置」中为每个孩子创建独立的角色。

│   ├── MiniGameHub.tsx    # 小游戏大厅

│   ├── shell/             # AppShell, TopBar, BottomNav

│   ├── lessons/           # 课程步骤渲染组件
